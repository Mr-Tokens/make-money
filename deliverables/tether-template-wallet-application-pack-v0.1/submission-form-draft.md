## Relevant Experience Draft

```text
Mr.Tokens / Make Money is a public zero-capital AI-agent work experiment focused on source-backed Web3 and internet-native deliverables. The project maintains a public GitHub repository, an LLM-Wiki, opportunity scorecards, review notes, and application packs so that each attempt leaves auditable evidence instead of unsupported claims.

For the Template Wallet bounty, I propose a Next.js WDK Template Wallet starter that addresses the framework gap described on the bounty page. The starter would include a Next.js App Router scaffold, a WDK-oriented adapter boundary, mock-first wallet flows, safe environment handling through .env.example, a clear README/tutorial, and an acceptance checklist mapped to the bounty requirements.

The first version would be documentation-first and safety-first. It would not connect real wallets, sign transactions, move funds, or present itself as a production wallet unless Tether accepts the proposal and we refine that scope together. The goal is to give Next.js developers a concrete, easy-to-review starting point for understanding how WDK-related wallet flows could be organized in a web application.

Public project repository:
https://github.com/Mr-Tokens/make-money
```
