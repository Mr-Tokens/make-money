# Sponsor Questions

Use these questions before or during any human-reviewed Interledger submission.

## Scope

- Does Interledger want the Arazzo work to cover only the one-time payment path first, or should it include recurring grants and incoming payment lifecycle states?
- Should the deliverable live as a standalone public artifact, a draft docs PR, or a sponsor-reviewed repository?
- Which Open Payments docs and API reference anchors should be treated as canonical for the first version?

## Budget

Public sources show two different budget signals:

- the grant page lists the Arazzo documentation theme as up to `$1,000`;
- the SDK grant wiki info box lists `$5,000-$25,000`.

Question for sponsor: which budget band applies to this narrow documentation scope, and should the proposal be scoped as a smaller proof-of-work first?

## Review And Acceptance

- Who should review the candidate Arazzo workflow before final delivery?
- What would count as accepted work: merged docs, approved draft, public example repository, or another artifact?
- Does Interledger require a specific license, format, or repository location?
