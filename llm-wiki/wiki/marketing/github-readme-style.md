# GitHub README Style Guide

Status: active
Date: 2026-05-20
Raw source: `llm-wiki/raw/sources/2026-05-20-github-trending-readme-style.md`

## Purpose

This guide gives the writing worker a reference style for public GitHub README drafts. It is based on current GitHub Trending README patterns, adapted for Make Money.

The goal is not to sound excited. The goal is to help a stranger understand the project, trust the process, and know what action to take next.

## Voice

Use an operator voice:

- concrete;
- direct;
- public-facing;
- evidence-aware;
- not self-narrating;
- not flattering the project owner;
- not promising returns.

Avoid:

- "I" as heroic narrator;
- authority-observer framing;
- "investment" language for wallet support;
- guaranteed income language;
- motivational filler;
- unsupported status claims.

## Public README Shape

Recommended order:

1. Project name.
2. One-sentence literal positioning.
3. Current status and revenue.
4. Short explanation of the experiment.
5. Proof links: LLM-Wiki, evidence ledger, strategy page.
6. How it works in 5 to 8 steps.
7. Current sprint or next deliverable.
8. Support/tip section with public wallet addresses.
9. Risk and non-financial-advice note.
10. Contribution or follow-along section.

## First Screen Rules

The first screen should answer:

- What is this?
- What is the current state?
- What is being tested?
- Where is the evidence?
- What can the reader do next?

For Make Money, "current revenue: 0" should stay visible until evidence changes it.

## Support Section Rules

Use this framing:

```text
Support this experiment

This is optional support for a public zero-capital experiment. It is not an investment, token sale, or promise of return. Public wallet addresses are listed so supporters can tip the project and so future records can be audited.
```

Do not write:

- "invest in this project";
- "early investor";
- "return";
- "profit share";
- "guaranteed";
- "risk-free".

## Proof Links

A strong README points to proof instead of repeating claims.

Use links to:

- `llm-wiki/wiki/current-status.md`;
- strategy records;
- experiment records;
- opportunity scorecards;
- submitted artifacts;
- public wallet addresses;
- future evidence ledger.

## Mimo Worker Instruction

When polishing a README:

1. Read the task brief.
2. Read this guide if provided as a source.
3. Preserve facts and current revenue status.
4. Improve scannability, headings, section order, and public clarity.
5. Replace hype with evidence or next action.
6. Return only the requested README draft unless asked for notes.
