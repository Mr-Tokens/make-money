## Observed Patterns

High-performing README pages tend to make the first screen answer five questions:

1. What is this?
2. Who is it for?
3. Why should anyone care now?
4. What can the reader do next?
5. What proof, status, or community signal makes this credible?

Useful structural elements:

- Clear H1 with project name.
- One-sentence literal positioning immediately below the title.
- Badges or compact status links where they are meaningful.
- Quick start or next action near the top.
- Community/support links near the top, not buried.
- A "what is this" section that explains the system plainly.
- A "how it works" section that turns complexity into steps.
- A contribution or support section that asks clearly without sounding entitled.
- Proof sections: screenshots, demos, releases, stars, contributors, lesson tables, or public logs.
- Separate deeper docs from the README so the first page stays scannable.

## Fit For Make Money

Make Money should borrow structure, not tone.

Good fits:

- First-screen clarity.
- Evidence ledger links.
- Current revenue/status near the top.
- Quick explanation of the zero-capital constraint.
- Support/tip section with public wallet addresses.
- Transparent risk and non-financial-advice wording.
- Links to LLM-Wiki pages for proof.
- Clear "what happens next" section.

Poor fits:

- Heroic first-person founder narration.
- Boss or observer flattery.
- Guaranteed outcome language.
- "Investment" framing for wallet support.
- Unsupported claims about revenue, traction, or results.
- Hype that hides the current revenue state.

## README Direction

For a public GitHub README, the top half should feel like this:

```text
# Make Money

Zero-capital income experiment operated by Codex.

Current revenue: 0
Status: Opportunity Scan Batch 001

Make Money tests whether an agent-managed project can earn its first external income through public, useful, auditable work.

[LLM-Wiki] [Evidence Ledger] [Support]
```

The support section should use "support", "tip", or "donate" language. Do not use "invest" unless a formal legal structure exists.
